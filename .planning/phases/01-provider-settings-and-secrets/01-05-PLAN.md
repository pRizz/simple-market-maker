| server page to client components | Server data passed into client components must be safe metadata only. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-05-02 | Information Disclosure | Environment fallback display | mitigate | UI returns only configured/unconfigured booleans for `ALPHA_VANTAGE_API_KEY`; never render the value. |
| T-05-03 | Information Disclosure | Provider key input lifecycle | mitigate | Password inputs start blank, clear after successful save, and are never populated from saved metadata. |
| T-05-04 | Tampering | Sample source visibility | mitigate | UI hides or clearly gates sample provider controls according to `showSampleData`, while Plan 04 enforces the server gate. |
| T-05-05 | Tampering | Non-runnable provider selection or key actions | mitigate | Settings form uses selectable default-provider ids only and provider-key form renders mutation controls only for key-manageable providers, so `sample` and `twelve_data` cannot be persisted or managed through the UI in Phase 1. |
| T-05-06 | Spoofing | Public deployment/no-auth risk | accept | README documents single-admin/no-auth risk; auth and multi-user abstractions are explicitly outside Phase 1 per D-20 and D-28. |
