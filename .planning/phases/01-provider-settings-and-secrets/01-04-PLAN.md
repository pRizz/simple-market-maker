|----------|-------------|
| settings resolver to market-data provider | Plaintext credential crosses between server-only modules. |
| market-data form/API to service | User-selected `source` can request sample or provider-backed data. |
| external provider to service | Provider failure details are untrusted before sanitization. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | `provider-factory.ts` credential handoff | mitigate | Resolve credentials server-side and pass the key only into `AlphaVantageMarketDataProvider`; never return resolver results to API/UI. |
| T-04-02 | Elevation of Privilege | Disabled saved key | mitigate | Rely on Plan 03 resolver `enabled` check before provider construction; provider factory tests cover disabled-key exclusion through resolver behavior. |
| T-04-03 | Tampering | Sample source bypassing UI gate | mitigate | Enforce `showSampleData` in `market-data-service.ts` before provider construction so API clients cannot bypass hidden sample settings. |
| T-04-04 | Information Disclosure | Provider fetch errors | mitigate | Sanitize provider errors before returning `formErrors`, with tests for raw URL, `apikey=`, and raw key removal. |
| T-04-05 | Denial of Service | Live provider quota consumption | accept | This phase keeps explicit fetch behavior only; request dedupe, progress, and quota orchestration are scoped to Phase 3. |
| T-04-06 | Information Disclosure | Ciphertext/IV/auth tag leakage | transfer | Plan 02 and Plan 03 keep encrypted material inside settings server repositories and API DTO tests; Plan 04 consumes only resolver output. |
