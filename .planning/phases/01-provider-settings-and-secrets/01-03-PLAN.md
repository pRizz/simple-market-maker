<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser to API routes | Untrusted JSON enters settings and provider-key mutation endpoints. |
| validation service to Alpha Vantage | Saved plaintext key is used in a third-party request. |
| server credential resolver to provider factory | Plaintext key stays server-only while selecting fetch credentials. |
| API routes to browser | JSON responses must contain only safe metadata and configured booleans. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Provider validation responses | mitigate | Sanitize raw URLs, `apikey=`, raw keys, and provider response bodies before storing or returning validation messages. |
| T-03-02 | Information Disclosure | Settings/provider-key API routes | mitigate | Return only safe metadata DTOs and fallback configured booleans; tests assert encrypted and plaintext fields are absent. |
| T-03-03 | Tampering | Settings and provider-key route inputs | mitigate | Parse raw JSON through domain/service validators and reject unsupported provider ids or invalid setting values with `400` or `404`. |
| T-03-04 | Elevation of Privilege | Disabled saved key | mitigate | Credential resolver checks `enabled === true` before decrypting or returning a persisted key. |
| T-03-05 | Information Disclosure | Missing encryption secret unsafe fallback | mitigate | Validation and resolver saved-key paths fail closed with safe messages when `PROVIDER_KEYS_ENCRYPTION_SECRET` is missing. |
| T-03-06 | Spoofing | Public deployment/no-auth risk | accept | Single-admin/no-auth is locked for this milestone; routes must remain documented as unsafe for public deployment until an auth phase exists. |
| T-03-07 | Information Disclosure | Ciphertext/IV/auth tag leakage | mitigate | API contract tests assert route responses do not include `encryptedKey`, `encryptionIv`, `encryptionAuthTag`, or database column equivalents. |
| T-03-08 | Tampering | Sample source bypassing server-side gate | transfer | Plan 04 owns `market-data-service.ts` sample gate enforcement; Plan 03 only exposes settings mutation. |
| T-03-09 | Tampering | Non-runnable providers selected or validated | mitigate | Settings API accepts only selectable default providers and provider-key routes accept only key-manageable providers, so `sample` and `twelve_data` are rejected until their later gates/adapters exist. |
</threat_model>
